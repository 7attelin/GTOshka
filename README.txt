Бот-помощник для сдачи нормативов

Telegram-бот для помощи студентам в подготовке и сдаче вузовских нормативов и нормативов ГТО. Разработан на Python 3.12+ с использованием библиотеки python-telegram-bot.

================================================================================
ВОЗМОЖНОСТИ
================================================================================

Вузовские нормативы:
- Нормативы СПО: Отправка фотографии с нормативами.
- Нормативы ВО: Отправка PDF-файла с полным списком нормативов, описание содержимого и пояснения для студентов разных медицинских групп.

Нормативы ГТО:
- Выбор возрастной ступени (6, 7, 8) с указанием возраста.
- Просмотр фотографии нормативов для выбранной ступени.
- Подробные рекомендации по недельной двигательной активности для каждой ступени.
- Прямые ссылки на официальные страницы ступеней на сайте ГТО.

Инфо:
- Ссылка на официальный сайт ГТО.
- Отправка таблицы (Excel) со ссылками на видео-уроки по технике выполнения упражнений (легкая атлетика, волейбол, баскетбол и др.).
- Умное обновление меню после отправки файла, чтобы избежать спама.

================================================================================
БЫСТРЫЙ СТАРТ
================================================================================

1. Установка зависимостей
-------------------------
Убедитесь, что у вас установлен Python 3.12 или выше.

pip install -r requirements.txt


2. Получение токена бота
------------------------
1. Откройте Telegram и найдите @BotFather (https://t.me/BotFather)
2. Отправьте команду /newbot
3. Придумайте имя и username для бота (username должен заканчиваться на bot)
4. Скопируйте полученный токен.


3. Настройка
------------
Создайте файл .env в корне проекта:

cp .env.example .env

Откройте .env и вставьте ваш токен:

TOKEN=ваш_токен_от_botfather


4. Подготовка файлов
--------------------
Бот ожидает наличие определенных папок и файлов для корректной работы.

Структура папок:
Все медиафайлы должны находиться в папке data относительно корня проекта.

- Изображения: data/images/
  Сюда помещаются фотографии нормативов (.jpg).
  Имена файлов должны соответствовать настройкам в config.py (по умолчанию: 
  spo_1.jpg, gto_stage_6.jpg, gto_stage_7.jpg, gto_stage_8.jpg).

- Документы: data/files/
  Сюда помещаются PDF-файлы и таблицы.
  По умолчанию бот ищет:
  * Контрольные и практические задания для ВО.pdf (нормативы ВО)
  * video_links_table.xlsx (таблица с видео-уроками для раздела "Инфо")

ВАЖНО: Если вы используете другие имена файлов, обязательно обновите словари 
IMAGES и FILES в файле config.py.


5. Запуск бота
--------------
python bot.py

При успешном запуске вы увидите сообщение: "Бот успешно запущен. Ожидание команд."

================================================================================
НАСТРОЙКА
================================================================================

Все тексты, ссылки и пути к файлам вынесены в файл config.py для удобного 
редактирования без изменения логики бота.

Основные параметры:
- INSTITUTE_SITE: ссылка на сайт вашего института.
- GTO_MAIN_SITE: основная ссылка на сайт ГТО.
- GTO_STAGE_LINKS: словарь со ссылками на конкретные ступени ГТО.
- GTO_RECOMMENDATIONS: подробные тексты рекомендаций для каждой ступени. 
  Поддерживает Markdown-форматирование.

Обновление путей к файлам:
Если вы изменили имена файлов в папках data/images или data/files, отредактируйте 
соответствующие словари в config.py:

IMAGES = {
    "spo_1": "data/images/spo_1.jpg",
    "gto_stage_6": "data/images/gto_stage_6.jpg",
    # ...
}

FILES = {
    "vo_norms_pdf": "data/files/Контрольные и практические задания для ВО.pdf",
    "video_links_table": "data/files/video_links_table.xlsx",
}

================================================================================
РАЗВЕРТЫВАНИЕ НА СЕРВЕРЕ
================================================================================

Требования для продакшена:
- ОС: Linux (Ubuntu/Debian/CentOS)
- Python 3.12+
- Постоянное подключение к интернету
- Менеджер процессов (systemd, supervisor) или Docker

Режимы работы:

1. Polling (рекомендуется для начала)
   Используется по умолчанию в bot.py. Бот сам опрашивает сервера Telegram на 
   наличие новых сообщений. Это самый простой способ запуска, не требующий 
   настройки домена и SSL-сертификатов.
   
   Для работы в фоне на сервере используйте systemd.

2. Webhook (для высоконагруженных систем)
   Режим вебхука требует наличия домена и SSL-сертификата. В данном проекте 
   по умолчанию настроен polling. Переключение на webhook требует изменения 
   функции main() в bot.py.

Настройка systemd сервиса (Linux):
Это рекомендуемый способ обеспечения постоянной работы бота на сервере.

1. Создайте файл сервиса:
   sudo nano /etc/systemd/system/gto-bot.service

2. Добавьте следующее содержимое (замените пути на актуальные):

   [Unit]
   Description=GTO Normativs Telegram Bot
   After=network.target

   [Service]
   Type=simple
   User=www-data
   Group=www-data
   WorkingDirectory=/home/user/GTOshka
   ExecStart=/usr/bin/python3.12 /home/user/GTOshka/bot.py
   Restart=always
   RestartSec=10

   [Install]
   WantedBy=multi-user.target

3. Активируйте и запустите сервис:
   sudo systemctl daemon-reload
   sudo systemctl enable gto-bot
   sudo systemctl start gto-bot

4. Проверьте статус:
   sudo systemctl status gto-bot

================================================================================
ОСОБЕННОСТИ НАВИГАЦИИ
================================================================================

Бот реализует гибридный подход к взаимодействию для удобства пользователя:

1. Меню: При навигации по меню сообщения РЕДАКТИРУЮТСЯ (обновляется текст и 
   клавиатура), чтобы не засорять чат.

2. Контент (Фото/Файлы): При запросе нормативов старое сообщение меню УДАЛЯЕТСЯ, 
   и бот отправляет новые сообщения с файлами/фотографиями.

3. Защита от спама: В разделе "Инфо" при запросе таблицы с видео сообщение меню 
   ОБНОВЛЯЕТСЯ (статус кнопки меняется), а файл повторно не отправляется, если 
   он уже был выслан в текущей сессии взаимодействия.
